# 【要件定義書】サイネージコンテンツクラフトAI

オフィスビル運営を支える、お知らせ作成・配信DXアプリケーション

---

## 1. プロダクト概要

「サイネージコンテンツクラフトAI」は、オフィスビルのプロパティマネジメント（PM）業務における3つの課題——「サイネージのコンテンツ不足」「お知らせ作成・複数媒体への投稿（マルチポスト）の負荷」「スケジュールの見えにくさ」——を、AIとデータベースの一元化によって解決するWebアプリケーションです。

管理者が情報を一度入力するだけで、AIが各媒体（サイネージ、メール、ポータル、PDF）に最適化したコンテンツを自動生成・配信し、あわせて館内スケジュールを可視化します。「一度の入力で、すべての媒体へ」を実現することがコンセプトです。

---

## 2. 対象ユーザーと課題

**対象ユーザー**

オフィスビルの管理会社、PM（プロパティマネジメント）担当者、ビルマネジメント会社。

**抱えている課題**

第一に、サイネージの有料コンテンツ（ニュース・天気など）を契約しているものの、同じ内容がループ再生されるだけで費用対効果が低く、独自コンテンツを作る余裕もない点。第二に、工事や点検のお知らせを、サイネージ（画像）・メール・入居者ポータル・掲示板（PDF）と、媒体ごとに手作業で個別に作成・加工しており、業務負荷が非常に重い点。第三に、各種イベントや工事が「単発のお知らせ」にとどまり、月間・年間のスケジュールとして俯瞰できない点です。

---

## 3. システム機能要件

### 機能①：AIサイネージコンテンツ自動生成

館内サイネージに流す独自コンテンツを、AIが自動で編成・生成する機能です。

- **イベント報告映像の自動配信（新規）**
  ビル内で実施したイベントの様子を伝える紹介テキストや映像を、実施後にサイネージ用として自動編成します。
- **プライバシー保護加工 / AI匿名化（新規）**
  館内カメラやイベント撮影の映像をサイネージに流す際、映像内の人物の顔や個人を特定できる情報をAIが自動検知します。ぼかし・モザイク・アバター化など「個人を特定できない加工」をリアルタイムまたは半自動で施し、プライバシーに配慮したうえでビルの賑わいを映像コンテンツとして配信できます。

### 機能②：ワンソース・マルチアウトプット（お知らせ一括生成・配信）

一度の入力データから、複数媒体向けのコンテンツをまとめて生成・配信する中核機能です。

- **簡易情報入力フォーム**
  管理者が工事・点検の内容を、フリーテキスト（一言）または所定の定型フォーマットで入力できる画面。
- **マルチメディア一括生成**
  一度の入力から、AIが裏側で以下の媒体用コンテンツを同時に生成します。
  - サイネージ用案内画像
  - テナント向け通知メール本文（テキスト）
  - 入居者ポータルサイト用の掲載テキスト
  - 館内掲示板用の案内（PDF出力レイアウト）
- **ワンクリック配信確認（エージェント機能）**
  生成された各媒体向けコンテンツを管理者が確認し、「このまま配信する」ボタンを押すだけで、各媒体への自動投稿・メール送信・PDFダウンロードが実行されます。

### 機能③：タイムライン・カレンダー自動可視化

入力された予定を、自動でカレンダー上に見える化する機能です。

- **カレンダー自動マッピング**
  機能②で入力・確定した工事・点検・イベントの予定が、共通データベースを通じてカレンダー画面（月間・週間ビュー）へ自動でプロットされます。
- **ダッシュボードでの俯瞰**
  管理者および（将来的には）入居者が、ビルで「いつ・何が起きるか」を一目で把握できるタイムライン・カレンダー画面を備えます。

### 機能④：防災・緊急時モード（BCP対策）

地震・台風・火災・近隣交通の遅延といった緊急時に、システム全体を切り替える機能です。

- **緊急ワンタップ切り替え**
  緊急事態の発生時、管理者が管理画面からワンタップ（または一言入力）するだけで、システム全体を「防災モード」へ切り替えます。
- **サイネージの緊急画面ジャック**
  配信中の通常コンテンツを強制中断し、避難経路・防災情報・近隣インフラ状況（運行情報など）を表示する緊急画面に切り替えます。
- **テナント向け緊急一斉連絡**
  サイネージの切り替えと同時に、登録済みのテナント担当者宛てに、状況説明やビル側の対応方針（例：「ロビーを開放します」など）を記載した緊急メールを、AIが自動作成・一斉送信します。

## 4. 画面構成案（モックアップ要件）

GitHubのIssueなどで画面イメージを議論するための基本構成案です。

**ダッシュボード（ホーム）**
今月・今週のビルスケジュール（カレンダー表示）、および現在サイネージに配信中のコンテンツプレビューを表示します。

**お知らせ作成・配信画面**
点検・工事情報の入力フォームに加え、AI生成結果のプレビュー領域（サイネージ画像／メール文／ポータル文／PDFイメージをタブで切り替えられる構成）と、「一斉配信・確定」ボタンを備えます。

**AIサイネージ自動配信 設定画面**
配信スケジュール設定（毎日・毎週・時間指定）と、生成コンテンツのテーマ・プロンプト管理（例：「近くのおすすめランチ情報」「季節の挨拶と防災の知恵」など）を行います。

## 5. 技術スタック・アーキテクチャの選定方針（開発メモ）

| 領域 | 候補・方針 |
|---|---|
| フロントエンド／バックエンド | 迅速なプロトタイプ開発が可能な構成（例：Next.js、FastAPI など） |
| AI・LLM連携（画像） | DALL-E 3、Midjourney API、またはテンプレートベースの画像合成エンジン |
| AI・LLM連携（テキスト） | OpenAI API（GPT-4o）、Anthropic Claude API など |
| インフラ・データベース | スケジュールとお知らせデータを一元管理するリレーショナルDB（PostgreSQL など） |
